# 微信小程序开发者工具 CLI MCP 设计方案

## 一、项目概述

### 1.1 目标

创建一个完整的 MCP (Model Context Protocol) 服务器，封装微信开发者工具 CLI 的所有功能，使 AI 助手能够通过自然语言调用小程序开发工具的各项能力。

### 1.2 功能范围

覆盖微信开发者工具 CLI 的**全部**功能：
- 登录管理
- 代码预览与上传
- 项目窗口管理
- 云开发操作
- 自动化测试
- 缓存管理
- npm 构建

## 二、CLI 功能映射

### 2.1 功能分类

| 分类 | CLI 命令 | MCP Tool 名称 | 说明 |
|------|----------|---------------|------|
| **登录** | `cli login` | `weapp_login` | 登录开发者工具 |
| | `cli islogin` | `weapp_check_login` | 检查登录状态 |
| **预览/上传** | `cli preview` | `weapp_preview` | 生成预览二维码 |
| | `cli auto-preview` | `weapp_auto_preview` | 自动预览 |
| | `cli upload` | `weapp_upload` | 上传代码 |
| **项目管理** | `cli open` | `weapp_open` | 打开工具/项目 |
| | `cli open-other` | `weapp_open_other` | 以"其他"模式打开 |
| | `cli close` | `weapp_close` | 关闭项目 |
| | `cli quit` | `weapp_quit` | 退出工具 |
| **构建** | `cli build-npm` | `weapp_build_npm` | 构建 npm |
| | `cli cache` | `weapp_clear_cache` | 清除缓存 |
| **自动化** | `cli auto` | `weapp_auto` | 开启自动化 |
| | `cli auto-replay` | `weapp_auto_replay` | 自动化测试回放 |
| **云开发** | `cli cloud env list` | `weapp_cloud_env_list` | 云环境列表 |
| | `cli cloud functions list` | `weapp_cloud_functions_list` | 云函数列表 |
| | `cli cloud functions info` | `weapp_cloud_functions_info` | 云函数信息 |
| | `cli cloud functions deploy` | `weapp_cloud_functions_deploy` | 部署云函数 |
| | `cli cloud functions inc-deploy` | `weapp_cloud_functions_inc_deploy` | 增量部署 |
| | `cli cloud functions download` | `weapp_cloud_functions_download` | 下载云函数 |
| **其他** | `cli reset-fileutils` | `weapp_reset_fileutils` | 重置文件监听 |
| | `cli engine build` | `weapp_engine_build` | 引擎构建 |

### 2.2 全局参数映射

所有工具共享以下通用参数：

| 参数 | 类型 | 环境变量 | 说明 |
|------|------|----------|------|
| `project` | string | - | 项目路径 |
| `appid` | string | - | 小程序 AppID |
| `extAppid` | string | - | 第三方平台 AppID |
| `port` | number | `WEAPP_PORT` | HTTP 服务端口 |
| `lang` | string | `WEAPP_LANG` | 语言 (en/zh) |
| `debug` | boolean | `WEAPP_DEBUG` | 调试模式 |

### 3.2 核心模块设计

#### 3.2.1 CLI 客户端 (`cli-client.ts`)

负责与微信开发者工具 CLI 交互：

```typescript
class WeappCLIClient {
  private cliPath: string;
  private defaultPort: number;
  
  // 执行 CLI 命令
  async execute(command: string, args: string[], options?: ExecuteOptions): Promise<CLIResult>;
  
  // 自动检测 CLI 路径
  private async detectCliPath(): Promise<string>;
  
  // 构建命令参数
  private buildArgs(params: Record<string, any>): string[];
}
```

#### 3.2.2 MCP 服务器 (`server.ts`)

MCP 协议实现：

```typescript
class WeappMCPServer {
  private server: Server;
  private cliClient: WeappCLIClient;
  
  // 注册所有工具
  private registerTools(): void;
  
  // 注册资源
  private registerResources(): void;
  
  // 注册提示符
  private registerPrompts(): void;
  
  // 启动服务器
  async start(): Promise<void>;
}
```

#### 3.2.3 工具注册中心 (`tools/index.ts`)

统一管理所有工具定义：

```typescript
export interface ToolDefinition {
```

## 七、错误处理策略

### 7.1 错误分类

| 错误类型 | 错误码 | 说明 |
|----------|--------|------|
| CLI 未找到 | 1001 | CLI 路径不存在或无法访问 |
| 端口未开启 | 1002 | 开发者工具服务端口未开启 |
| 未登录 | 1003 | 需要先登录微信开发者工具 |
| 项目无效 | 1004 | 项目路径或配置无效 |
| 执行超时 | 1005 | CLI 命令执行超时 |
| 参数错误 | 1006 | 输入参数验证失败 |

### 7.2 错误响应格式

```typescript
{
  content: [{
    type: "text",
    text: JSON.stringify({
      success: false,
      error: {
        code: 1003,
        message: "请先登录微信开发者工具",
        suggestion: "使用 weapp_login 工具进行登录"
      }
    }, null, 2)
  }],
  isError: true
}
```
